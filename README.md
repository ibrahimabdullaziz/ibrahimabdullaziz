##  Engineering Philosophy

> I don't just write code that works — I engineer systems that last.

My C/C++ background gave me something most frontend engineers lack: a genuine understanding of what happens underneath the abstractions. I apply that thinking to every layer of the stack — from React render cycles to PostgreSQL query plans to API throughput under load.

I believe in:
- **Depth over breadth** — master the fundamentals, frameworks become easy
- **Shipping over perfecting** — real feedback beats imagined edge cases
- **Documentation as craft** — if it's not written down, it didn't happen
- **Learning in public** — writing about what I build makes it stick
